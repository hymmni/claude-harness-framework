# Robot Research Harness Guide

본 가이드는 AI를 활용하여 로봇 행동 지능 프로젝트를 효율적으로 개발하는 방법을 설명합니다.

## 1. 프로젝트 초기화
1. `projects/` 폴더에 베이스라인 코드를 `git clone` 합니다.
2. `references/` 폴더에 참고할 논문 코드를 `git clone` 합니다.
3. 클로드에게 활성 프로젝트를 알립니다: "현재 `projects/my_diffusion_policy`에서 작업 중이야."

## 2. 참조 및 이식 워크플로우
- **분석**: "references/에 있는 오픈소스에서 시계열 데이터를 전처리하는 부분을 분석해줘."
- **이식**: "분석한 내용을 바탕으로 projects/ 내부의 베이스라인 코드에 적용해줘. reference 출처는 주석으로 남겨줘."
- **기록**: 수정이 끝나면 `experiments/` 폴더에 로그를 작성하게 합니다.

## 3. 3-PC 워크플로우 (하드웨어 분리)
현재 연구 환경은 다음과 같이 분리되어 운영됩니다:
1. **코딩용 PC (Local)**: harness_framework를 통해 클로드와 함께 코드를 수정하고 \`git push\`를 수행합니다. (GPU 없음)
2. **학습용 PC (Server)**: 원격으로 \`git pull\`을 받아 학습을 돌리고 WandB로 결과를 관찰합니다.
3. **추론용 PC (Robot)**: 학습된 체크포인트를 받아 실제 로봇 환경에서 구동합니다.

## 4. 유의 사항
- **데이터 관리**: 대용량 데이터는 `.gitignore`에 의해 제외됩니다. 데이터셋은 심볼릭 링크(ln -s)를 사용하여 외부 경로와 연결하는 것을 권장합니다.
- **환경 분리 (Environment Isolation)**: 하네스 레포는 코딩 템플릿일 뿐입니다. 가상환경 세팅(Conda yml 등)은 항상 \`projects/<active_project>\` 내부에서 독립적으로 관리하십시오. 다른 PC(학습/추론)에서는 하네스 전체가 아닌 해당 프로젝트 레포만 클론하여 사용하기 때문입니다.
